Q：你对 Java 集合了解有多少？

A：（咳咳嗓子说道）Java的集合体系主要是包括了 Collection接口 和 Map接口。

​ 其中父Collection接口是最基本的集合接口，一个Collection代表一组元素，还继承了Iterable接口，Iterable 是用于遍历集合类的标准访问方法。Iterable 中提供了Iterator 定义了遍历集合的方法：hasNext（）/ next / remove 等。

​ Collection 它不提供该接口的具体实现。具体的子接口有 List 和 Set 接口 和 Queue 接口。

​ Queue 接口在集合的基础上添加了增删改查操作，并且队列默认使用FIFO（先进先出）规则，实现该接口的主要有：LinkedList 可以用它来实现双向队列。 PriorityQueue 基于堆结构实现，可以用它来实现优先队列。

​ List 接口是有序可重复的，实现该接口的有 ArrayList（基于动态数组实现，支持随机访问，扩容1.5倍。） Vector 和 ArrayList 类似，但它是线程安全的，默认扩容一倍。Stack: 继承自 Vector，实现栈。LinkedList：基于双向链表实现，只能顺序访问，但是可以快速地在链表中间插入和删除元素。不仅如此，LinkedList 还可以用作栈、队列和双向队列。

​ Set 接口的元素无序,不可重复 (没有索引)，具体的实现类有诸如基于红黑树实现的 TreeSet，支持有序性操作，例如根据一个范围查找元素的操作。但是查找效率不如 HashSet，基于哈希表实现，内部是value为空的 HashMap，支持快速查找，但不支持有序性操作。并且失去了元素的插入顺序信息。HashSet 查找的时间复杂度为 O(1)，TreeSet 则为 O(logN)

​ Map 接口的特点是无序，存放的是键值对；key部分不能重复。实现该接口的类主要有 WeakHashMap： 改进的HashMap，实现了“弱引用”，如果一个key不被引用，则被GC回收。基于红黑树实现的TreeMap是有序的。基于哈希表实现的HashMap。 HashTable：和 HashMap 类似，但它是线程安全的。LinkedHashMap：使用双向链表来维护元素的顺序，顺序为插入顺序或者最近最少使用（LRU）顺序。

​ 刚才说到的集合大多都是线程不安全的，除了少部分是线程安全的，如 HashTable ，Vector 这些，在实现方法的内部加了synchronized关键字，使得多线程的环境下，只有一个线程能获得该方法的锁，只有获得锁的线程才能执行该方法。

​ 如果想使用线程安全的容器类，可以去用 juc 包下的容器类。如 ConcurrentHashMap 等
